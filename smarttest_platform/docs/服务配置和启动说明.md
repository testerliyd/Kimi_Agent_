# SmartTest 智能化测试平台服务配置和启动说明

## 一、环境准备

### 1.1 系统要求

| 组件 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 20GB | 50GB+ |
| 操作系统 | Linux/Windows/Mac | Ubuntu 22.04 LTS |

### 1.2 依赖软件

- Python 3.11+
- MySQL 8.0+
- Redis 7.0+
- Nginx (生产环境推荐)

## 二、安装步骤

### 2.1 安装Python环境

#### Ubuntu/Debian
```bash
# 安装Python3.11
sudo apt update
sudo apt install -y python3.11 python3.11-venv python3.11-dev

# 安装pip
curl https://bootstrap.pypa.io/get-pip.py | python3.11

# 验证安装
python3.11 --version
pip --version
```

#### CentOS/RHEL
```bash
# 安装Python3.11
sudo yum install -y centos-release-scl
sudo yum install -y rh-python311
scl enable rh-python311 bash

# 验证安装
python3.11 --version
```

#### Windows
1. 下载Python3.11安装包: https://www.python.org/downloads/
2. 运行安装程序，勾选"Add Python to PATH"
3. 验证安装: `python --version`

### 2.2 安装MySQL

#### Ubuntu/Debian
```bash
# 安装MySQL
sudo apt update
sudo apt install -y mysql-server-8.0

# 启动MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation

# 创建数据库
sudo mysql -u root -p
```

```sql
CREATE DATABASE smarttest CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'smarttest'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON smarttest.* TO 'smarttest'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

#### Docker安装MySQL
```bash
docker run -d \
  --name smarttest_mysql \
  -e MYSQL_ROOT_PASSWORD=your_password \
  -e MYSQL_DATABASE=smarttest \
  -p 3306:3306 \
  -v mysql_data:/var/lib/mysql \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

### 2.3 安装Redis

#### Ubuntu/Debian
```bash
sudo apt update
sudo apt install -y redis-server

# 启动Redis
sudo systemctl start redis
sudo systemctl enable redis

# 验证安装
redis-cli ping
```

#### Docker安装Redis
```bash
docker run -d \
  --name smarttest_redis \
  -p 6379:6379 \
  -v redis_data:/data \
  redis:7-alpine
```

## 三、项目部署

### 3.1 方式一：手动部署

#### 1. 克隆项目
```bash
cd /opt
git clone <repository-url> smarttest
cd smarttest
```

#### 2. 创建虚拟环境
```bash
python3.11 -m venv venv
source venv/bin/activate
```

#### 3. 安装依赖
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

#### 4. 配置环境变量
```bash
cp .env.example .env
vim .env
```

修改以下配置:
```bash
DJANGO_SECRET_KEY=your-random-secret-key
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=your-domain.com,localhost

DB_NAME=smarttest
DB_USER=smarttest
DB_PASSWORD=your-password
DB_HOST=localhost
DB_PORT=3306

REDIS_URL=redis://localhost:6379/1
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0
```

#### 5. 数据库迁移
```bash
python manage.py migrate
```

#### 6. 初始化数据
```bash
python init_db.py
```

#### 7. 创建超级用户
```bash
python manage.py createsuperuser
```

#### 8. 收集静态文件
```bash
python manage.py collectstatic --noinput
```

#### 9. 启动服务

**开发环境**
```bash
python manage.py runserver 0.0.0.0:8000
```

**生产环境（使用Gunicorn）**
```bash
# 安装Gunicorn
pip install gunicorn

# 启动服务
gunicorn smarttest.wsgi:application \
  --bind 0.0.0.0:8000 \
  --workers 4 \
  --timeout 120 \
  --access-logfile logs/access.log \
  --error-logfile logs/error.log
```

#### 10. 启动Celery

**Worker**
```bash
celery -A smarttest worker -l info --concurrency=4
```

**Beat（定时任务）**
```bash
celery -A smarttest beat -l info
```

### 3.2 方式二：Docker Compose部署

#### 1. 安装Docker和Docker Compose
```bash
# Ubuntu
curl -fsSL https://get.docker.com | sh

sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

#### 5. 初始化数据

```

#### 6. 创建超级用户
```bash
docker-compose exec web python manage.py createsuperuser
```

#### 7. 查看日志
```bash
docker-compose logs -f
```

#### 8. 停止服务
```bash

```

## 四、Nginx配置（生产环境）

### 4.1 安装Nginx
```bash
sudo apt install -y nginx
```

### 4.2 配置文件
```bash
sudo vim /etc/nginx/sites-available/smarttest
```

```nginx
upstream smarttest {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://smarttest;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /static/ {
        alias /opt/smarttest/staticfiles/;
    }

    location /media/ {
        alias /opt/smarttest/media/;
    }
}
```

### 4.3 启用配置
```bash
sudo ln -s /etc/nginx/sites-available/smarttest /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 4.4 HTTPS配置（使用Certbot）
```bash
# 安装Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo systemctl enable certbot.timer
```

## 五、系统服务配置

### 5.1 创建Systemd服务

#### Gunicorn服务
```bash
sudo vim /etc/systemd/system/smarttest.service
```

```ini
[Unit]
Description=SmartTest Django Application
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/smarttest
Environment=DJANGO_SETTINGS_MODULE=smarttest.settings
EnvironmentFile=/opt/smarttest/.env
ExecStart=/opt/smarttest/venv/bin/gunicorn smarttest.wsgi:application --bind 0.0.0.0:8000 --workers 4
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

#### Celery Worker服务
```bash
sudo vim /etc/systemd/system/smarttest-celery.service
```

```ini
[Unit]
Description=SmartTest Celery Worker
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/smarttest
Environment=DJANGO_SETTINGS_MODULE=smarttest.settings
EnvironmentFile=/opt/smarttest/.env
ExecStart=/opt/smarttest/venv/bin/celery -A smarttest worker -l info
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

#### Celery Beat服务
```bash
sudo vim /etc/systemd/system/smarttest-beat.service
```

```ini
[Unit]
Description=SmartTest Celery Beat
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/smarttest
Environment=DJANGO_SETTINGS_MODULE=smarttest.settings
EnvironmentFile=/opt/smarttest/.env
ExecStart=/opt/smarttest/venv/bin/celery -A smarttest beat -l info
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

### 5.2 启动服务
```bash
# 重载配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start smarttest
sudo systemctl start smarttest-celery
sudo systemctl start smarttest-beat

# 设置开机自启
sudo systemctl enable smarttest
sudo systemctl enable smarttest-celery
sudo systemctl enable smarttest-beat

# 查看状态
sudo systemctl status smarttest
```

## 七、备份和恢复

### 7.1 数据库备份

```bash
# 手动备份
mysqldump -u smarttest -p smarttest > backup_$(date +%Y%m%d).sql

# 自动备份脚本
#!/bin/bash
BACKUP_DIR="/opt/backups"
DB_NAME="smarttest"
DB_USER="smarttest"
DB_PASS="your_password"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR
mysqldump -u $DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/smarttest_$DATE.sql.gz

# 保留最近30天备份
find $BACKUP_DIR -name "smarttest_*.sql.gz" -mtime +30 -delete
```

### 7.2 数据库恢复

```bash
# 恢复备份
mysql -u smarttest -p smarttest < backup_20240101.sql

# 恢复压缩备份
gunzip < backup_20240101.sql.gz | mysql -u smarttest -p smarttest
```

## 八、常见问题

### Q1: 数据库连接失败
```bash
# 检查MySQL状态
sudo systemctl status mysql

# 检查连接
mysql -u smarttest -p -h localhost smarttest

# 检查防火墙
sudo ufw status
```

### Q2: Redis连接失败
```bash
# 检查Redis状态
sudo systemctl status redis

# 测试连接
redis-cli ping
```

### Q3: 静态文件加载失败
```bash
# 重新收集静态文件
python manage.py collectstatic --noinput --clear

# 检查权限
sudo chown -R www-data:www-data /opt/smarttest/staticfiles
```

### Q4: Celery任务不执行
```bash
# 检查Celery状态
sudo systemctl status smarttest-celery

# 查看日志
sudo journalctl -u smarttest-celery -f

# 重启Celery
sudo systemctl restart smarttest-celery
```
