## 1. 系统登录

### 1.1 首次登录
1. 访问系统地址: `http://localhost:8000`
2. 使用管理员账号登录
3. 首次登录后建议修改密码

### 1.2 获取API Token
```bash
curl -X POST http://localhost:8000/api/auth/token/ \
  -H "Content-Type: application/json" \
  -d '{"username": "your_username", "password": "your_password"}'
```

响应示例:
```json
{
  "access": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "refresh": "eyJ0eXAiOiJKV1QiLCJhbGc..."
}
```

### 1.3 Token刷新
```bash
curl -X POST http://localhost:8000/api/auth/token/refresh/ \
  -H "Content-Type: application/json" \
  -d '{"refresh": "your_refresh_token"}'
```

## 2. 项目管理

### 2.1 创建项目
1. 登录系统后，点击左侧菜单"项目管理"
2. 点击"新建项目"按钮
3. 填写项目信息:
   - 项目编码: 系统自动生成，也可手动输入
   - 项目名称: 必填
   - 项目类型: Web应用/移动应用/API服务等
   - 优先级: 低/中/高/紧急
   - 描述: 项目详细描述
   - 时间规划: 开始日期和结束日期
4. 点击"保存"

### 2.2 添加项目成员
1. 进入项目详情页
2. 点击"成员管理"标签
3. 点击"添加成员"按钮
4. 选择用户和角色:
   - 所有者: 拥有所有权限
   - 管理员: 项目管理权限
   - 测试工程师: 测试相关权限
   - 开发工程师: 开发和Bug相关权限
   - 访客: 只读权限
5. 点击"确认"

### 2.3 创建版本
1. 进入项目详情页
2. 点击"版本管理"标签
3. 点击"新建版本"按钮
4. 填写版本信息:
   - 版本号: 如 v1.0.0
   - 版本名称: 如 "春季发布"
   - 计划开始/结束日期
   - 版本范围描述
5. 点击"保存"

### 2.4 配置测试环境
1. 进入项目详情页
2. 点击"环境配置"标签
3. 点击"新建环境"按钮
4. 填写环境信息:
   - 环境名称: 如 "测试环境"
   - 环境类型: 开发/测试/预发布/生产
   - 基础URL: 如 `https://api-test.example.com`
   - 环境变量: JSON格式，如 `{"api_key": "test_key"}`
5. 点击"保存"

## 3. 用例管理

### 3.1 创建测试用例
1. 点击左侧菜单"用例管理"
2. 选择项目，点击"新建用例"按钮
3. 填写用例信息:
   - 用例编号: 系统自动生成
   - 用例名称: 必填
   - 用例类型: 功能/性能/安全/兼容性等
   - 优先级: P0-P3
   - 所属模块: 如 "用户管理"
   - 前置条件: 执行用例前的准备条件
   - 测试步骤: JSON格式
     ```json
     [
       {"step": 1, "action": "打开登录页面", "expected": "页面正常加载"},
       {"step": 2, "action": "输入用户名和密码", "expected": "输入成功"}
     ]
     ```
   - 预期结果: 用例执行的预期结果
4. 点击"保存"

### 3.2 执行测试用例
#### 单条执行
1. 进入用例详情页
2. 点击"执行"按钮
3. 选择执行结果:
   - 通过: 测试通过
   - 失败: 测试失败
   - 阻塞: 因其他问题无法执行
   - 跳过: 跳过此用例
4. 填写实际结果（可选）
5. 点击"确认"

#### 批量执行
1. 在用例列表页，选择多个用例
2. 点击"批量执行"按钮
3. 选择执行结果
4. 点击"确认"

### 3.3 创建测试套件
1. 点击"测试套件"标签
2. 点击"新建套件"按钮
3. 填写套件名称和描述
4. 保存后，点击"添加用例"
5. 选择要添加的用例

### 3.4 创建测试计划
1. 点击"测试计划"标签
2. 点击"新建计划"按钮
3. 填写计划信息:
   - 计划名称
   - 关联版本
   - 计划开始/结束时间
   - 负责人
4. 选择包含的测试套件
5. 点击"保存"

## 4. Bug管理

### 4.1 创建Bug
1. 点击左侧菜单"Bug管理"
2. 点击"新建Bug"按钮
3. 填写Bug信息:
   - 标题: 简洁描述问题
   - 详细描述: 问题详细说明
   - Bug类型: 功能/UI/性能等
   - 严重程度: 致命/严重/一般/轻微
   - 优先级: 紧急/高/中/低
   - 所属模块
   - 发现版本
   - 环境信息: 浏览器/操作系统等
   - 复现步骤
   - 预期结果和实际结果
   - 附件: 截图或日志
4. 点击"保存"

### 4.2 分配Bug
1. 进入Bug详情页
2. 点击"分配"按钮
3. 选择处理人
4. 点击"确认"

### 4.3 解决Bug
1. 进入Bug详情页
2. 点击"解决"按钮
3. 填写解决方案
4. 选择解决版本
5. 点击"确认"

### 4.4 关闭Bug
1. 进入已解决的Bug详情页
2. 点击"关闭"按钮
3. 确认关闭

### 4.5 添加评论
1. 进入Bug详情页
2. 在评论区输入内容
3. 可选择是否为内部评论（仅项目成员可见）
4. 点击"发送"

## 5. 接口自动化测试

### 5.1 创建API定义
1. 点击左侧菜单"接口测试"
2. 点击"API定义"标签
3. 点击"新建API"按钮
4. 填写API信息:
   - API名称
   - 请求方法: GET/POST/PUT/DELETE等
   - 请求路径: 如 `/api/users`
   - 所属环境
   - 请求头: JSON格式
   - URL参数: JSON格式
   - Content-Type
   - 请求体: JSON格式或原始文本
5. 点击"保存"

### 5.2 快速测试API
1. 进入API详情页
2. 点击"快速测试"按钮
3. 可覆盖请求参数
4. 点击"发送请求"
5. 查看响应结果

### 5.3 创建API测试用例
1. 点击"测试用例"标签
2. 点击"新建用例"按钮
3. 选择API定义
4. 填写用例信息:
   - 用例名称
   - 覆盖请求参数（可选）
   - 预期状态码
   - 预期响应体（可选）
   - 验证规则（JSON Schema）
5. 点击"保存"

### 5.4 创建测试任务
1. 点击"测试任务"标签
2. 点击"新建任务"按钮
3. 填写任务信息:
   - 任务名称
   - 选择测试用例（可多选）
   - 选择执行环境
4. 点击"保存"

### 5.5 执行测试任务
1. 进入任务详情页
2. 点击"执行"按钮
3. 等待执行完成
4. 查看执行结果和报告

### 5.6 发送测试报告到飞书
1. 执行完成后，点击"发送报告"
2. 选择飞书群聊（或输入群聊ID）
3. 点击"发送"

## 6. 性能测试

### 6.1 创建测试场景
1. 点击左侧菜单"性能测试"
2. 点击"测试场景"标签
3. 点击"新建场景"按钮
4. 填写场景信息:
   - 场景名称
   - 场景类型: 负载/压力/尖峰/耐力/可扩展性
   - 目标URL
   - HTTP方法
   - 请求头/请求体
   - 并发用户数
   - 启动时间（渐进式增加用户）
   - 测试时长
   - 成功标准（如平均响应时间<1000ms）
5. 点击"保存"

### 6.2 创建测试任务
1. 点击"测试任务"标签
2. 点击"新建任务"按钮
3. 选择测试场景
4. 可覆盖测试参数
5. 点击"保存"

### 6.3 执行性能测试
1. 进入任务详情页
2. 点击"执行"按钮
3. 等待测试完成（可在执行过程中查看实时指标）
4. 查看测试报告

### 6.4 查看性能指标
1. 进入任务详情页
2. 点击"性能指标"标签
3. 查看各项指标:
   - 响应时间（平均/最小/最大/P50/P95/P99）
   - 吞吐量（RPS/BPS）
   - 错误率
   - 并发用户数曲线

## 8. 飞书集成

平台支持两种方式配置飞书机器人：

### 方式一：Webhook方式（推荐，简单快捷）

这种方式适合快速接入，无需创建飞书应用。

#### 8.1.1 获取Webhook地址和密钥
1. 在飞书群聊中点击"设置" → "群机器人"
2. 点击"添加机器人"，选择"自定义机器人"
3. 设置机器人名称和描述
4. **安全设置**（推荐开启签名校验）:
   - 勾选"签名校验"
   - 保存生成的密钥（Encrypt Key）
5. 复制Webhook地址

#### 8.1.2 配置Webhook
1. 点击左侧菜单"飞书集成"
2. 点击"飞书配置"标签
3. 点击"新建配置"按钮
4. 填写配置信息:
   - 配置名称: 如"测试群机器人"
   - **Webhook URL**: 填入复制的Webhook地址
   - **Encrypt Key**: 填入签名校验密钥（如果开启了签名校验）
   - 其他字段留空
5. 点击"保存"
6. 点击"测试连接"验证配置

### 方式二：飞书应用方式（功能更完整）

这种方式需要创建飞书应用，功能更完整。

#### 8.2.1 创建飞书应用
1. 访问[飞书开放平台](https://open.feishu.cn/)
2. 登录并进入"开发者后台"
3. 点击"创建企业自建应用"
4. 填写应用名称和描述
5. 在"凭证与基础信息"中获取:
   - App ID
   - App Secret

#### 8.2.2 配置应用权限
1. 进入应用详情页
2. 点击"权限管理"
3. 添加以下权限:
   - `im:chat:readonly`（获取群聊信息）
   - `im:message:send`（发送消息）
   - `im:message.group_msg`（发送群消息）
4. 点击"申请发布" → "创建版本"
5. 提交审核（企业内部应用可直接发布）

#### 8.2.3 配置平台
1. 点击左侧菜单"飞书集成"
2. 点击"飞书配置"标签
3. 点击"新建配置"按钮
4. 填写配置信息:
   - 配置名称
   - **App ID**: 填入应用凭证
   - **App Secret**: 填入应用密钥
   - 默认群聊ID（可选）
5. 点击"保存"
6. 点击"测试连接"验证配置

### 8.3 绑定项目群聊
1. 进入项目详情页
2. 点击"飞书绑定"标签
3. 点击"绑定群聊"按钮
4. 选择飞书配置
5. 输入群聊ID
6. 配置通知开关:
   - Bug创建通知
   - Bug状态变更通知
   - 测试完成通知
7. 点击"保存"

### 8.3 发送测试消息
1. 点击"发送消息"按钮
2. 选择消息类型（文本/卡片）
3. 输入消息内容
4. 选择目标群聊
5. 点击"发送"

## 10. 常见问题

### Q1: 如何重置密码？
A: 在登录页面点击"忘记密码"，或联系管理员重置。

### Q2: API测试执行失败怎么办？
A: 
1. 检查API定义是否正确
2. 检查环境配置是否正确
3. 查看详细错误信息
4. 检查网络连接

### Q3: 飞书通知收不到？
A:
1. 检查飞书配置是否正确
2. **Webhook方式**:
   - 确认Webhook URL正确
   - 如果开启了签名校验，确认Encrypt Key正确
   - 在群聊中@机器人，确认机器人已添加到群聊
3. **应用方式**:
   - 确认App ID和App Secret正确
   - 确认应用有发送消息权限
   - 检查群聊ID是否正确
4. 查看通知日志排查问题

### Q3.1: 如何获取飞书Webhook地址？
A:
1. 在飞书群聊中点击"设置" → "群机器人"
2. 点击"添加机器人"，选择"自定义机器人"
3. 设置完成后即可看到Webhook地址
4. 建议开启"签名校验"以提高安全性

### Q3.2: 飞书Webhook签名校验失败？
A:
1. 确认Encrypt Key正确（注意区分大小写）
2. 确认系统时间与标准时间同步（误差不超过1小时）
3. 可以使用`test_feishu_webhook.py`脚本测试签名生成

### Q4: 大模型连接失败？
A:
1. 检查API Key是否正确
2. 检查网络连接
3. 确认账户有余额
4. 尝试更换基础URL
